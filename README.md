# Golem Gateway (AI Core)

Модульный API-маршрутизатор без сохранения состояния (stateless) для больших языковых моделей. Golem Gateway работает как невидимый мост между веб-интерфейсами искусственного интеллекта и стандартными API-клиентами. Он предоставляет единый REST-интерфейс, совместимый со стандартом OpenAI, используя автоматизацию headless-браузеров (Puppeteer) и перехват XHR/Fetch запросов.

## 🌟 Архитектура и возможности

- **Полная совместимость с OpenAI API:** Нативная поддержка эндпоинтов `/v1/models` и `/v1/chat/completions` (включая потоковую передачу `stream: true`). Работает прямо "из коробки" с любым клиентом (SillyTavern, Оболочки кодеров и др.).
- **Панель Управления (Dashboard):** Красивый и современный веб-интерфейс (доступен на порту `7777`) для быстрого захвата токенов сессий, а также визуального управления ядром.
- **Динамическое управление памятью:** Встроенная система настроек позволяет включать и отключать модули нейросетей на лету. Ненужные адаптеры выгружаются из оперативной памяти одним кликом без перезагрузки сервера.
- **Автоматическая стерилизация истории:** Теневые сессии на целевых облачных платформах (DeepSeek, Qwen и др.) удаляются сразу после завершения генерации ответа, сохраняя ваш аккаунт в идеальной чистоте.
- **Модульная система провайдеров:** Базируется на паттерне Router. Логика ядра строго изолирована от реализаций конкретных нейросетей (папка `providers/`). Спроектировано так, чтобы добавление любой новой нейросети занимало минимум времени.

## 📦 Требования

- [Node.js](https://nodejs.org/) (рекомендуется версия 16.0.0 или выше)
- Установленный Google Chrome или Chromium на хост-машине.

## 🚀 Установка и запуск

### Запуск в один клик (Windows)
Для пользователей Windows предусмотрен максимальный комфорт. Вам **не нужно** вручную устанавливать библиотеки или вводить команды в консоль:
1. Склонируйте репозиторий или скачайте исходный код в zip-архиве и распакуйте его.
2. Дважды кликните по файлу **`start.bat`**.
> Скрипт полностью автономен: он сам проверит наличие Node.js модулей, скачает их через `npm install` и запустит ядро маршрутизатора. Откроется интерфейс управления.

### Linux / macOS (Вручную)
1. Склонируйте репозиторий в нужную директорию.
2. Откройте терминал и установите зависимости:
 ```bash
 npm install
 ```
3. Запустите сервер:
 ```bash
 node index.js
 ```

*Примечание: При первом запуске ядро автоматически откроет локальную Панель Управления (`http://127.0.0.1:7777`) в вашем браузере для авторизации провайдеров.*

## 🔌 Интеграция с программным обеспечением (SillyTavern и др.)

Настройте ваше клиентское приложение (например, SillyTavern) выбрав тип API **`OpenAI Custom Endpoint`** (или совместимый), указав следующий адрес базы:
```text
http://127.0.0.1:7777/v1
```
Ключ API не требуется (введите любую строку, например `sk-dummy`). Все доступные модели из активных адаптеров подтянутся автоматически при нажатии кнопки обновления.

**Управление системными тегами:**
Некоторые модули могут возвращать служебную информацию (теги мыслей `<think>`, блоки веб-поиска и др.). Если вы хотите исключить их из долгосрочной памяти персонажа в RP-клиентах, используйте встроенные инструменты регулярных выражений (Regex) вашего клиента, чтобы скрывать или заменять эти конструкции на пустую строку.

## 🛠 Разработка: Добавление новых провайдеров

Система спроектирована как открытый шлюз. Чтобы добавить новый ИИ:
1. Создайте новый скрипт внутри директории `providers/` (например, `providers/claude.js`).
2. Реализуйте и экспортируйте 4 стандартные функции жизненного цикла: `initProvider()`, `setupRoutes()`, `handleChatCompletion()` и `unloadProvider()`.
3. Добавьте провайдера в массив в файле `providers.js` (указав иконку и payload для перехвата токенов).
4. Добавьте правила маршрутизации в `index.js` (блок `/v1/chat/completions`) и тумблер в логику сохранения `settings.js`.
